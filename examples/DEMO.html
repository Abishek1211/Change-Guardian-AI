🔬 How ChangeGuardian Analyzes Changes

1️⃣ Intake Agent
Parses the change request and extracts service name, old/new values, and parameters using regex patterns.
2️⃣ Scenario Router
Classifies the change type: framework_upgrade, resource_change, db_schema, api_contract, shared_dependency, or event_schema.
3️⃣ Graph Impact Agent
Uses NetworkX to traverse the dependency graph and find all affected services, APIs, databases, and Kafka events.
4️⃣ Hybrid RAG Agent
Vector RAG: FAISS searches similar past incidents
Vectorless RAG: Exact rule checking (Java version, memory safety, null constraints, etc.)
5️⃣ Memory Graph Agent
Looks up prior deployment outcomes for the same service or similar change type to learn from history.
6️⃣ Risk & Rollout Agent
Deterministic scoring (0-100) based on service criticality, affected count, incidents, rule violations, and memory graph failures.
7️⃣ LLM Explanation Agent
Local Qwen2.5 (via Ollama on AMD hardware) generates plain-English risk explanation and 3 concrete remediation steps. Falls back to rule-based summaries if LLM is offline.
