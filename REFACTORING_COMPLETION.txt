1. DYNAMIC MODEL SELECTION
   Automatically selects best model based on available RAM:
   - qwen2.5:3b (6GB VRAM, 16GB RAM)
   - qwen2.5:7b (12GB VRAM, 32GB RAM)
   - qwen2.5:14b (20GB VRAM, 64GB RAM)
   - qwen3:30b-a3b (35GB VRAM, 128GB RAM) ← NEW
   - llama3.1:70b (60GB VRAM, 256GB RAM) ← NEW

2. EXPERT LLM PROMPTS
   Sophisticated prompts for larger models:
   - SYSTEM_PROMPT_EXPERT for 30B+ models
   - SYSTEM_PROMPT_STANDARD for smaller models
   - Auto-selects based on model reasoning_depth

3. FINANCIAL IMPACT TRACKING
   Report includes financial impact from similar incidents:
   - $500K if Spring Boot migration fails
   - $250K if OOMKill happens
   - $750K if library upgrade breaks everything

4. SLA-AWARE RISK ASSESSMENT
   Considers SLA commitment in risk scoring:
   - "CRITICAL SLA RISK" for 99.99% services
   - "SLA AT RISK" for 99.0%+ services
   - "SLA ACCEPTABLE" for lower SLAs

5. STAGING TEST AWARENESS
   Memory graph tracks if changes were tested:
   - [Tested in Staging] - increased confidence
   - [NOT tested] - additional risk flag

6. PROFESSIONAL GRADIO UI
   Interactive web interface with:
   - Color-coded risk gauges

   - Incident severity badges
   - Affected service tags
